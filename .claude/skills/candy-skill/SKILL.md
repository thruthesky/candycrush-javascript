# 🍬 캔디크러시 게임 스킬

## 개요

VanillaScript(순수 JavaScript)와 HTML, CSS만으로 캔디크러시(매치3 퍼즐) 게임을 구현하는 스킬이다.
외부 라이브러리 없이 브라우저 기본 API만 사용한다.

### 핵심 특징

| 항목 | 설명 |
|------|------|
| 🍭 이모지 캔디 | 6종 이모지를 캔디로 사용 |
| 🔊 효과음 | Web Audio API로 매치/스왑/콤보 효과음 |
| ✨ 애니메이션 | CSS keyframes + transitions로 풍부한 시각 효과 |
| 📱 반응형 | 모바일/데스크톱 모두 지원 |

## 프로젝트 파일 구조

```
index.html          # 메인 HTML (게임 진입점)
css/
  style.css         # 게임 스타일, 애니메이션, 레이아웃
js/
  game.js           # 게임 핵심 로직 (보드, 매칭, 스왑, 점수)
  audio.js          # 효과음 시스템 (Web Audio API)
```

## 구현 워크플로우

다음 순서로 구현할 것:

1. **HTML 구조 생성** → `index.html`에 게임 보드 컨테이너, 점수판, UI 배치
2. **CSS 스타일링** → 그리드 레이아웃, 캔디 셀, 애니메이션 정의
3. **게임 보드 초기화** → 8×8 그리드에 랜덤 이모지 캔디 배치
4. **매칭 로직 구현** → 가로/세로 3개 이상 매치 탐지 및 제거
5. **스왑 메커니즘** → 클릭 기반 인접 캔디 교환 (유효하지 않으면 되돌림)
6. **캐스케이드 처리** → 매치 제거 후 중력 낙하 + 새 캔디 생성 + 연쇄 매치
7. **효과음 추가** → Web Audio API로 매치/스왑/콤보/게임오버 사운드
8. **점수 및 UI** → 점수 계산, 콤보 배율, 레벨 시스템, 게임오버 조건

## 이모지 캔디 종류

```javascript
const CANDIES = ['🍬', '🍭', '🍫', '🍩', '🍪', '🧁'];
```

각 캔디는 고유 색상 배경과 함께 표시된다. 특수 캔디(4매치, 5매치 등)도 지원:

| 매치 수 | 특수 캔디 | 이모지 | 효과 |
|---------|----------|--------|------|
| 4개 매치 | 줄무늬 캔디 | 💥 | 한 줄 전체 제거 |
| 5개 매치 | 컬러 폭탄 | 🌈 | 같은 종류 캔디 모두 제거 |
| L/T 매치 | 폭탄 캔디 | 💣 | 3×3 영역 제거 |

## 참조 문서

### 게임 보드 및 캔디 시스템 → [game-board.md](references/game-board.md)

게임 보드 초기화, 8×8 그리드 생성, 캔디 배치 알고리즘을 다룬다.
HTML에서 CSS Grid로 보드를 렌더링하고, JavaScript로 2차원 배열 기반
게임 상태를 관리하는 방법을 설명한다. 초기 보드 생성 시 3매치가
발생하지 않도록 방지하는 로직, 캔디 셀의 DOM 생성 및 이벤트
바인딩, 보드 상태와 DOM 동기화 패턴을 포함한다. `createBoard()`,
`renderBoard()`, `getCandy()` 등 핵심 함수의 소스코드와
CSS Grid 레이아웃 코드를 상세히 기술한다.

### 매칭 및 캐스케이드 로직 → [matching-logic.md](references/matching-logic.md)

캔디크러시의 핵심인 매치3 탐지 알고리즘, 스왑 메커니즘, 캐스케이드
처리를 다룬다. 가로/세로 방향으로 3개 이상 연속된 같은 캔디를
탐지하는 `findMatches()` 함수, 인접 셀 간 캔디를 교환하는
`swapCandies()` 함수, 매치 제거 후 빈 공간에 위의 캔디를
떨어뜨리고 새 캔디를 생성하는 `cascadeDrop()` 함수를 포함한다.
특수 캔디 생성 조건(4매치→줄무늬, 5매치→컬러폭탄, L/T→폭탄)과
연쇄 매치 처리 루프의 소스코드를 상세히 기술한다.

### 애니메이션 효과 → [animations.md](references/animations.md)

CSS @keyframes와 transitions를 활용한 풍부한 시각 효과를 다룬다.
캔디 스왑 시 부드러운 이동 애니메이션, 매치 시 폭발/반짝임 효과,
새 캔디 낙하 시 바운스 애니메이션, 점수 획득 시 팝업 텍스트
애니메이션을 포함한다. JavaScript에서 CSS 클래스를 동적으로
추가/제거하여 애니메이션을 트리거하는 패턴, `animationend` 이벤트
활용법, 파티클 효과 구현 방법을 소스코드와 함께 설명한다.
콤보 시 화면 흔들림, 특수 캔디 활성화 시 빛 번짐 효과도 포함한다.

### 효과음 시스템 → [sound-effects.md](references/sound-effects.md)

Web Audio API를 사용한 효과음 생성 및 재생 시스템을 다룬다.
외부 오디오 파일 없이 OscillatorNode와 GainNode로 프로그래매틱하게
사운드를 합성하는 방법을 설명한다. 매치 시 "띵" 소리, 스왑 시
"슉" 소리, 콤보 시 상승 음계, 특수 캔디 폭발 시 "퍅" 소리,
게임오버 시 하강 멜로디를 각각의 주파수/파형/지속시간 파라미터와
함께 소스코드로 기술한다. `AudioManager` 클래스의 전체 구현과
음소거 토글 기능을 포함한다.

## 핵심 구현 규칙

1. **프레임워크 금지** — React, Vue 등 사용하지 않는다. 순수 VanillaScript만 사용
2. **단일 HTML 진입점** — `index.html`이 CSS/JS를 로드하는 구조
3. **이모지 우선** — 캔디는 반드시 이모지로 표현. 이미지 파일 사용 금지
4. **사운드 합성** — 오디오 파일 없이 Web Audio API로 사운드 합성
5. **애니메이션 CSS 우선** — 가능한 한 CSS 애니메이션 사용. JS 애니메이션은 보조적으로만
6. **모바일 지원** — touch 이벤트와 mouse 이벤트 모두 처리
7. **게임 루프** — `requestAnimationFrame` 대신 이벤트 기반 처리 (매치3는 턴제)

## 점수 시스템

```
기본 점수: 매치당 10점 × 매치된 캔디 수
콤보 배율: 연쇄 횟수 × 1.5배
특수 캔디 보너스:
  - 줄무늬(💥): +50점
  - 폭탄(💣): +100점
  - 컬러폭탄(🌈): +200점
```
